# Database

## Overview

PostgreSQL 16, accessed via SQLx (`sqlx` crate, async, compile-time checked queries). All primary keys use UUIDv7 (time-sortable, generated by `uuid` crate).

## Migration 002 (`002_map_columns.sql`)

```sql
ALTER TABLE communities ADD COLUMN map_community_id UUID;
ALTER TABLE communities ADD COLUMN map_secret_key BYTEA;
```

These columns link a Komun community to a piggPin map relay community. The `map_secret_key` is the X25519 secret key for that relay community, stored on the server for operational use (DEK wrapping, member key distribution).

## Migration conventions

1. **Sequential numbering**: `001_*.sql`, `002_*.sql`, `003_*.sql`, etc.
2. **Additive only**: New migrations add columns/tables. Never edit existing migrations.
3. **Descriptive names**: `001_schema.sql`, `002_map_columns.sql`.
4. **No down migrations**: There's no rollback mechanism. Think carefully before adding.
5. **Run automatically**: SQLx runs all pending migrations on `main()` startup via `sqlx::migrate!("../../migrations").run(&pool).await`.

## SQLx conventions

### Compile-time checked queries
SQLx macros (`sqlx::query!`, `sqlx::query_as!`, `sqlx::query_scalar!`) check SQL syntax against the database at compile time. This requires a live database connection during `cargo build`.

### Offline mode (`SQLX_OFFLINE=true`)
For Docker builds and CI where no database is available, SQLx uses a pre-generated offline query cache. After adding or changing queries, run:

```bash
cargo sqlx prepare
```

This updates `.sqlx/query-*.json` files used by the offline mode.

### Query patterns
All database queries live in `crates/server/src/db/` modules:
- `db/communities.rs` — community queries
- `db/conversations.rs` — match and message queries
- `db/notifications.rs` — notification queries
- `db/posts.rs` — post queries

### Connection pool
Created in `main.rs`:
```rust
PgPoolOptions::new()
    .max_connections(config.database.max_connections)
    .connect(&config.database.url)
    .await
```
Configurable via `[database].max_connections` (default: 20).
